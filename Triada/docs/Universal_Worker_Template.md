# Техническое задание: Universal Worker Template

## Назначение

Universal Worker — универсальный исполнитель задач внутри AI Office.

Worker не принимает стратегических решений.

Worker не выполняет планирование.

Worker не управляет системой.

Worker выполняет только те задачи, которые назначены Оркестратором.

---

# Роль Worker

Worker является исполнителем.

Оркестратор является руководителем.

Принцип:

```text
Оркестратор думает.
Worker выполняет.
```

---

# Обязанности Worker

Worker обязан:

1. Получить задачу.
2. Проверить параметры задачи.
3. Загрузить необходимые навыки.
4. Загрузить необходимый контекст.
5. Выполнить задачу.
6. Сформировать отчёт.
7. Вернуть результат Оркестратору.
8. Перейти в состояние ожидания.

---

# Ограничения

Worker запрещено:

* самостоятельно менять задачи;
* самостоятельно менять приоритеты;
* самостоятельно выбирать стратегию проекта;
* самостоятельно запускать необратимые действия;
* самостоятельно менять инфраструктуру;
* выполнять действия вне назначенной задачи.

Все спорные ситуации передаются Оркестратору.

---

# Входящие данные

Worker получает структуру:

```yaml
task:
  id: task_001

  title: Найти конкурентов

  priority: normal

  model: deepseek-reasoner

  skills:
    - web_search
    - competitor_analysis

  tools:
    - github
    - notion

  context:
    project: AI Office

  instructions:
    - Найти конкурентов
    - Собрать данные
    - Подготовить отчёт

  expected_output:
    markdown_report
```

---

# Обработка задачи

Алгоритм:

```text
Получить задачу
        ↓

Проверить структуру
        ↓

Проверить навыки
        ↓

Загрузить навыки
        ↓

Загрузить инструменты
        ↓

Выполнить задачу
        ↓

Сформировать отчёт
        ↓

Вернуть результат
        ↓

Освободить память
        ↓

Ожидание новой задачи
```

---

# Система навыков

Worker не хранит навыки постоянно.

Навыки подгружаются только при необходимости.

Структура:

```text
skills/
├── web_search/
│   └── SKILL.md
│
├── github_analysis/
│   └── SKILL.md
│
├── report_generation/
│   └── SKILL.md
│
└── architecture_review/
    └── SKILL.md
```

---

# Загрузка навыков

При старте задачи Worker:

1. Получает список навыков.
2. Проверяет наличие навыков.
3. Загружает только нужные навыки.
4. Выполняет задачу.
5. Выгружает навыки после завершения.

Запрещено загружать все навыки системы одновременно.

---

# Работа с моделями

Worker использует только модель, назначенную Оркестратором.

Пример:

```yaml
model: deepseek-chat
```

или

```yaml
model: deepseek-reasoner
```

или

```yaml
model: claude-sonnet
```

Worker не меняет модель самостоятельно.

---

# Работа с инструментами

Worker получает доступ только к инструментам, указанным в задаче.

Пример:

```yaml
tools:
  - github
  - notion
```

Если инструмент не указан — использовать его запрещено.

---

# Отчёт Worker

После завершения задачи Worker обязан вернуть отчёт.

Формат:

```yaml
task_id: task_001

status: completed

skills_used:
  - web_search
  - competitor_analysis

result_summary: >
  Найдено 5 конкурентов.
  Подготовлен сравнительный анализ.

files_created:
  - report.md

warnings: []

recommendations:
  - Провести дополнительный анализ цен.
```

---

# Обработка ошибок

Если задача не может быть выполнена:

```yaml
task_id: task_001

status: failed

error_type: missing_skill

error_message: >
  Навык competitor_analysis не найден.
```

Ошибка передаётся Оркестратору.

Worker не принимает решение самостоятельно.

---

# Состояния Worker

Допустимые состояния:

```text
idle
loading
working
waiting
completed
failed
```

---

# Масштабирование

Все Workers являются экземплярами одного шаблона.

Пример:

```text
Worker-1
Worker-2
Worker-3
Worker-4
Worker-5
```

Все используют одинаковый код.

Изменяются только:

* задача;
* модель;
* навыки;
* инструменты;
* контекст.

---

# Отказоустойчивость

Если Worker завершился аварийно:

```text
Worker
    ↓
Ошибка
    ↓
Task Queue
    ↓
Возврат задачи
    ↓
Другой Worker
```

Задача не должна теряться.

---

# Главный принцип

Worker — это универсальный исполнитель.

Компетенции находятся в Skills.

Управление находится в Оркестраторе.

Worker должен оставаться максимально простым, лёгким и взаимозаменяемым.
