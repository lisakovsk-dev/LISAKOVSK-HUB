# Инструкция по анализу GitHub

1. Изучи структуру репозитория.
2. Проанализируй README.md для понимания назначения проекта.
3. Проверь активность разработки (последние коммиты, issues, pull requests).
4. Оцени качество кода и архитектурные решения.
5. Сделай выводы о применимости или состоянии проекта.
